এখানে **SWR** এবং **ISR**-এর বাংলায় সংক্ষিপ্ত ও সহজ ব্যাখ্যা দেওয়া হলো:

---

## 🔄 ১. **SWR (ক্লায়েন্ট-সাইড ডাটা ফেচিং)**

**কখন ব্যবহার করবেন?**  
যখন রিয়েল-টাইম ডাটা দরকার (যেমন: কার্ট, ইউজার প্রোফাইল, লাইভ ইনভেন্টরি)।

### ✅ উদাহরণ:
```jsx
'use client';
import useSWR from 'swr';

const fetcher = (url) => fetch(url).then(res => res.json());

export default function ProductPage() {
  const { data, error, isLoading } = useSWR('/api/product/123', fetcher);

  if (isLoading) return <div>লোড হচ্ছে...</div>;
  if (error) return <div>এরর!</div>;

  return <h1>{data?.name}</h1>;
}
```

> 💡 **SWR মানে = Stale-While-Revalidate**  
> - আগের ডাটা দেখায় (stale)  
> - ব্যাকগ্রাউন্ডে নতুন ডাটা আনে  
> - ফোকাস/কানেক্ট করলে নিজে নিজে আপডেট হয়

---

## 🕒 ২. **ISR (ইনক্রিমেন্টাল স্ট্যাটিক রিজেনারেশন)**

**কখন ব্যবহার করবেন?**  
যে পাবলিক পেজ মাঝে মাঝে আপডেট হয় (যেমন: প্রোডাক্ট লিস্ট, ব্লগ পোস্ট)।

### ✅ উদাহরণ:
```jsx
// app/products/[id]/page.js
export const revalidate = 3600; // ১ ঘণ্টা পর রিজেনারেট হবে

export default async function ProductPage({ params }) {
  const res = await fetch(`http://laravel/api/shop/products/${params.id}`);
  const product = await res.json();

  return <h1>{product.data.name}</h1>;
}
```

> 💡 **ISR কীভাবে কাজ করে?**  
> - পেজটা **স্ট্যাটিক HTML** হিসেবে তৈরি হয়  
> - CDN থেকে দ্রুত লোড হয়  
> - নির্দিষ্ট সময় পর পর ব্যাকগ্রাউন্ডে আপডেট হয়

---

## 🆚 কখন কোনটা ব্যবহার করবেন?

| কাজ | ব্যবহার করুন |
|----|------------|
| প্রোডাক্ট ডিটেইল পেজ (সবার জন্য) | ✅ **ISR** |
| কার্ট, ইউজার ড্যাশবোর্ড, অর্ডার হিস্ট্রি | ✅ **SWR** |
| হোমপেজ, ক্যাটাগরি পেজ | ✅ **ISR** |
| লাইভ স্টক/ইনভেন্টরি | ✅ **SWR** |

---

## 💡 একসাথেও ব্যবহার করতে পারেন:

- **ISR** দিয়ে স্ট্যাটিক HTML (নাম, দাম, বিবরণ)
- **SWR** দিয়ে লাইভ স্টক দেখাবেন

---

✅ **সংক্ষেপে:**  
- **ISR** = দ্রুত লোড, এসইও ফ্রেন্ডলি স্ট্যাটিক পেজ  
- **SWR** = রিয়েল-টাইম ক্লায়েন্ট সাইড ডাটা
