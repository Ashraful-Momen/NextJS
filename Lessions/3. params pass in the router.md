# **Next.js App Router – Dynamic Route Params (Simple Note)**

### **1️⃣ Folder Structure**

| URL                   | Folder/File Path                   |
| --------------------- | ---------------------------------- |
| `/dashboard/user`     | `app/dashboard/user/page.jsx`      |
| `/dashboard/user/:id` | `app/dashboard/user/[id]/page.jsx` |

* `[id]` → **dynamic route parameter**.

---

### **2️⃣ Accessing Params (Next.js 16+)**

* In **Server Components**, `params` is **async**.

```tsx
// app/dashboard/user/[id]/page.jsx
const UserDetails = async ({ params }) => {
  const { id } = await params; // unwrap async params
  return <h1>User ID: {id}</h1>;
};

export default UserDetails;
```

* **Do not use `'use client'`** if you just need params.

---

### **3️⃣ Client Component Usage (Optional)**

* If you need hooks/state:

```tsx
// Server wrapper
const UserDetails = async ({ params }) => {
  const { id } = await params;
  return <UserClient id={id} />;
};
export default UserDetails;

// Client component
'use client';
const UserClient = ({ id }) => <h1>User ID: {id}</h1>;
export default UserClient;
```

---

### **4️⃣ Link Navigation**

```tsx
import Link from 'next/link';

// Navigate to user 3
<Link href="/dashboard/user/3">User 3</Link>
```

---

### **5️⃣ Key Points**

1. `[param]` → dynamic route.
2. `params` is **async** in Next.js 16+ App Router.
3. Use `await params` in **server components**.
4. Use **client components** only if you need hooks/state.
5. URL `/dashboard/user/123` → `params.id = "123"`

---

### **6️⃣ Quick ASCII Map**

```
/dashboard/user          -> page.jsx
/dashboard/user/[id]     -> [id]/page.jsx
URL example:
 /dashboard/user/3       -> params.id = "3"
 /dashboard/user/abc     -> params.id = "abc"
```
