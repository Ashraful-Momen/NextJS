```typescript
import type { Metadata } from "next";
import "./globals.css";

export const metadata: Metadata = {
  title: "Create Next App",
  description: "Generated by create next app",
};

export default function RootLayout({
  children // will be a page or nested layout
}) {
  return (
    <html lang="en">
      <header>Header</header>
      <body>{children}</body>
      <footer>Footer</footer>
    </html>
  );
}
;
```

## Import Statement
```typescript
import type { Metadata } from "next";
```
- Imports the `Metadata` **type** (TypeScript-only) from Next.js
- Used for type-checking your page metadata (title, description, etc.)

## CSS Import
```typescript
import "./globals.css";
```
- Imports global styles that apply to **all** pages in your app
- CSS rules here affect every component

## Metadata Export
```typescript
export const metadata: Metadata = {
  title: "Create Next App",
  description: "Generated by create next app",
};
```
- Defines SEO/metadata for **all pages** that don't override it
- Sets browser tab title and meta description
- Can be overridden by individual pages

## RootLayout Component
```typescript
export default function RootLayout({
  children,
}: Readonly<{
  children: React.ReactNode;
}>)
```

**Key points:**
- **Required** in every Next.js App Router project
- Wraps **all pages** in your application
- `children` prop receives the page content
- `Readonly` ensures props can't be mutated

## Return JSX
```typescript
return (
  <html lang="en">
    <body>{children}</body>
  </html>
);
```
- Renders the root HTML structure
- `lang="en"` sets page language (important for accessibility/SEO)
- `children` injects the current page content

## Why This Matters

**This layout is special because:**
1. **Must** have `<html>` and `<body>` tags (unlike regular components)
2. Cannot be nested (only one root layout per app)
3. Renders once and persists across page navigation
4. Great for:
   - Global navigation menus
   - Footer that stays consistent
   - Theme providers
   - Analytics scripts

## Example Enhancement
```typescript
export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        <nav>Global Navigation</nav>  {/* Shows on all pages */}
        <main>{children}</main>       {/* Page content changes */}
        <footer>Global Footer</footer> {/* Shows on all pages */}
      </body>
    </html>
  );
}
```

This is the **foundation** of your entire Next.js application!
