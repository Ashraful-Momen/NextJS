# **Next.js 16+ App Router – Layout Concept Note**

---

## **1️⃣ What is a Layout?**

* A **layout** is a React component that wraps all pages **inside a folder**.
* Allows **shared UI** like headers, navbars, sidebars, or footers.
* Automatically applied to **all nested routes**.

---

## **2️⃣ Folder-Based Layout**

```
app/
├─ layout.jsx            → Root layout (applies to entire app)
├─ page.jsx              → Home page
└─ dashboard/
   ├─ layout.jsx         → Dashboard layout (applies to dashboard pages)
   ├─ page.jsx           → Dashboard home
   └─ user/
      ├─ page.jsx        → User Dashboard
      └─ [id]/
         └─ page.jsx     → User Details
```

* **Root layout** wraps everything.
* **Nested layout** wraps pages in its folder (and subfolders).

---

## **3️⃣ Root Layout Example (`app/layout.jsx`)**

```tsx
export default function RootLayout({ children }) {
  return (
    <html>
      <body>
        <header>
          <h1>My App</h1>
        </header>
        <main>{children}</main>
        <footer>© 2025 My App</footer>
      </body>
    </html>
  );
}
```

* Applies to **all pages** in the app.
* Can include **global nav, footer, CSS wrappers**.

---

## **4️⃣ Nested Layout Example (`app/dashboard/layout.jsx`)**

```tsx
export default function DashboardLayout({ children }) {
  return (
    <div>
      <nav>
        <a href="/dashboard">Home</a> | <a href="/dashboard/user">Users</a>
      </nav>
      <section>{children}</section>
    </div>
  );
}
```

* Wraps **all pages inside `/dashboard`**.
* Can include **dashboard-specific navbars or sidebars**.

---

## **5️⃣ Key Concepts**

1. **Layouts are per folder** → all nested pages inherit them.
2. **Root layout** → global shared UI.
3. **Nested layout** → folder-specific UI.
4. Can be **nested multiple levels** for more complex apps.
5. Works automatically; no extra import needed in pages.

---

## **6️⃣ ASCII Map for Clarity**

```
app/
├─ layout.jsx        -> Root layout (wraps all pages)
/                        |
├─ dashboard/
│   ├─ layout.jsx    -> Dashboard layout (wraps dashboard pages)
│   ├─ page.jsx      -> Dashboard home
│   └─ user/
│       ├─ page.jsx  -> User Dashboard
│       └─ [id]/page.jsx -> User Details
```

* **Root layout** wraps everything.
* **Dashboard layout** wraps `/dashboard` and all nested routes.
