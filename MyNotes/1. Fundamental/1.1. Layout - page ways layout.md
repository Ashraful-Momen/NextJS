# 🌍 1. Global Layout (`app/layout.js`)

```jsx id="f9x2pd"
// app/layout.js

/*
🔍 Root Layout Responsibility:

- Only defines <html> and <body>
- SHOULD NOT contain fixed UI like header/footer if you want flexibility
*/
export default function RootLayout({ children }) {
  return (
    <html>
      <body>
        {children}
      </body>
    </html>
  )
}
```

# 🏠 2. Main Layout (Default Header/Footer)

```jsx id="c7m4zs"
// app/(main)/layout.js
import MainHeader from '../components/MainHeader'
import MainFooter from '../components/MainFooter'

export default function MainLayout({ children }) {
  return (
    <>
      <MainHeader />
      <main>{children}</main>
      <MainFooter />
    </>
  )
}
```

# 📊 3. Dashboard Layout (Different Header/Footer)

# ⚠️ Core Insight (Custom Header/Footer Per Page)

```id="t5j8vn"
RootLayout (HTML only)
   ↓
Route Group Layout (decides Header/Footer)
   ↓
Page
```

---

# 🧠 Problem: “Different header/footer per page?”

### ❌ Don’t do this in RootLayout:

* Hardcoding Navbar/Footer
* Trying to conditionally hide everywhere

---

# ✅ Correct Solution (Best Practice)

👉 Move Header/Footer into **route group layouts**

### Why?

* Each section controls its own UI
* Clean separation (auth / dashboard / public)
* Scalable for large apps (like your microservice UI)

---

# 🔄 Alternative (Conditional Rendering — Not Recommended)

```jsx id="p1z6xc"
// app/layout.js
import { usePathname } from 'next/navigation'

export default function RootLayout({ children }) {
  const pathname = usePathname()

  const isDashboard = pathname.startsWith('/dashboard')

  return (
    <html>
      <body>
        {isDashboard ? <DashboardHeader /> : <MainHeader />}

        {children}

        {isDashboard ? <DashboardFooter /> : <MainFooter />}
      </body>
    </html>
  )
}
```

⚠️ Issues:

* Tight coupling
* Hard to scale
* Breaks separation of concerns

---

# ⚡ Key Concepts (Quick)

* Root layout → only structure (`html`, `body`)
* Route group layout → controls Header/Footer
* Different UI = different layout file
* `(folder)` → does NOT affect URL
* Best practice → split by concern (main, dashboard, auth)

---

# 🧠 Mental Model

```id="r2c7mw"
Route Group decides layout

(main)       → MainHeader + Footer
(dashboard)  → DashboardHeader + Sidebar + Footer
(auth)       → No Header/Footer
```
