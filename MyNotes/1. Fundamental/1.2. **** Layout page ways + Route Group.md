# 🧩 Next.js Different Layout Per Page (Using Route Groups)

* ✅ Global root layout (mandatory)
* ✅ Completely different layouts per page/section
* ✅ Separate Header/Footer per layout
* ✅ Clean, scalable structure

# 🌍 1. Root Layout (`app/layout.tsx`)

```tsx id="j9x4pt"
// app/layout.tsx

/*
🔍 RootLayout Rules:

- ALWAYS required
- Wraps entire app
- SHOULD contain ONLY html + body
- DO NOT put header/footer here if you want different layouts
*/
export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        {children}
      </body>
    </html>
  )
}
```

---

# 🏠 2. Main Layout (Default Site)

```tsx id="u7c2mw"
// app/(main)/layout.tsx
import MainHeader from '../components/MainHeader'
import MainFooter from '../components/MainFooter'

/*
🔍 Main Layout:

- Applies to all routes inside (main)
- Example: "/", "/about"
*/
export default function MainLayout({ children }: { children: React.ReactNode }) {
  return (
    <>
      <MainHeader />

      <main>{children}</main>

      <MainFooter />
    </>
  )
}
```

---

# 📝 3. Blog Layout (Completely Different)

```tsx id="x5r8kq"
// app/(blog)/layout.tsx
import BlogHeader from '../components/BlogHeader'
import BlogFooter from '../components/BlogFooter'

/*
🔍 Blog Layout:

- Applies ONLY to blog routes
- Completely separate UI from main layout
*/
export default function BlogLayout({ children }: { children: React.ReactNode }) {
  return (
    <>
      <BlogHeader />

      <section>{children}</section>

      <BlogFooter />
    </>
  )
}
```

---

# 🔐 4. Auth Layout (No Header/Footer)

```tsx id="p2m9zs"
// app/(auth)/layout.tsx

/*
🔍 Auth Layout:

- No header/footer
- Clean minimal UI
*/
export default function AuthLayout({ children }: { children: React.ReactNode }) {
  return (
    <div style={{ maxWidth: '400px', margin: 'auto' }}>
      {children}
    </div>
  )
}
```

# 🧠 Key Insight

✔ Layouts are **NOT shared across route groups**
✔ Each route group has its **own independent layout tree**
✔ Only **RootLayout is common**

---

# ⚡ Key Concepts (Quick)

* `layout.tsx` → controls UI wrapper
* `(group)` → route group (not in URL)
* Each group = separate layout system
* Root layout = global base only
* Best practice → keep header/footer inside group layouts

---

# 🚀 Final Rule (Remember This)

```id="t6k9wb"
Different UI (header/footer)
→ Different Route Group
→ Different layout.tsx
```
