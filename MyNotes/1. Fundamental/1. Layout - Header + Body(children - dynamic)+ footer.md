# рЯМН 1. Global Layout (`app/layout.js`)

This wraps **every page automatically**.

```jsx
// app/layout.js
import './globals.css'
import Navbar from './components/Navbar'
import Footer from './components/Footer'

export const metadata = {
  title: 'My App',
  description: 'Simple Next.js App',
}

/*
рЯФН RootLayout Explanation:

- RootLayout is the TOP-LEVEL layout of your entire app.
- Next.js automatically calls this function for EVERY route.
- It must return <html> and <body> (required in App Router).
- Think of it like a global wrapper (similar to _app.js in old Pages Router).
*/
export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>
        <Navbar />

        <main style={{ padding: '20px' }}>
          {children}
        </main>

        <Footer />
      </body>
    </html>
  )
}

/*
рЯФН children Explanation:

- "children" is a special React prop.
- It represents the CURRENT page or nested layout being rendered.

Examples:
- Visiting "/" вЖТ children = <HomePage />
- Visiting "/dashboard" вЖТ children = <DashboardLayout><DashboardPage /></DashboardLayout>

Execution Flow:
RootLayout
  вЖТ DashboardLayout (if exists)
    вЖТ Page (page.js)

So:
{children} = dynamic content injected inside the layout.
*/
```

```jsx
      </aside>

      <section style={{ flex: 1, padding: '20px' }}>
        {children}
      </section>
    </div>
  )
}
```
